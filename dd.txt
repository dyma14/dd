1. Як виконати дію закомічених змін на сервер?
git push -u origin master
2. Як виконати затягування із змінним сервером? 
git клон
3. Як синхронізація робочої копії із затягнутими змінами? 
git злиття
4. Як синхронізація робочої копії із затягнутими змінами для 
проект з під модулями?
git злиття
5. Де зберігаються файли при використанні розподіленої системи контролю 
версія? 
Усі файли містяться на кожному пк
6. Які переваги дають використання розподіленої системи контролю версії?
Кожен працює зі своїм репозиторієм
7. Які недоліки має використання розподіленої системи контролю версії ? 
Відсутність авторизації, нумерації
8. Що таке репозиторій?
Місце для зберігання проектів, сайтів з кодами
9. Навіщо потрібні розподілені системи?
Для спільної роботи над проектами разом з кільком людям
10.Що таке оновлення робочої копії проекту? 
Коли інші користувачі обновлюють робочу копію
11.Що таке фіксація змін проекту? 
Результат зліття двох або більше гілок
12.Що таке модифікація проекту? 
Зміни в існуючому проекті
13.Для чого використовує розгалуження в СКВ? 
Використовується для написання незакінчених версій
14.Що таке «Злиття версій»?
Злиття гілок в одну та зі змінами всіх гілок
15.Що розуміють під конфліктом при роботі СКВ? 
Коли один користувач додав або зміни файлу, це інші зміни, які змінилися в той час, коли буде конфлікт
16.За яких умов зліття версія проходить автоматично і без конфліктів?
Якщо не буде змін локальних файлів зліття відбудеться. Якщо ні то буде конфлікт
